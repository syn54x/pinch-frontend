# pinch-frontend

The Pinch web app: a Vite + React SPA over the [pinch-backend](../pinch-backend) API.

## Stack

TypeScript (strict) · TanStack Router (file-based) + TanStack Query · Tailwind + shadcn/ui · Biome · pnpm

The API client under `src/api/generated/` is generated by `@hey-api/openapi-ts` from the committed `openapi.json` snapshot — the contract seam with the backend. Never hand-edit it; run `just openapi-sync` to re-export from the backend and regenerate. CI fails when the generated client drifts from the snapshot.

## Development

```sh
just setup          # install dependencies
just dev            # dev server on :5173 (backend expected on :8000)
just check          # biome + tsc
just test           # vitest (quiet in F1 by design)
just e2e            # Playwright against a real backend on a fresh database
just openapi-sync   # refresh snapshot + regenerate the client
```

E2e uses dedicated ports (backend :8100, frontend :5183) and never touches your dev servers. It needs the `local-pg` docker container running.
